# 1. 회의 핵심 방향

이번 회의의 핵심은 기존 **LLM 기반 감정분류 시스템**을 발전시켜,

```text
긍정/부정 분류
↓
세부 감정 분류
↓
감정 score 정량화 (0~1)
↓
감정 분포(Emotional Distribution)
↓
케어 키트 추천
```

으로 확장하는 것이다.

즉, 단순히

```text
긍정 / 부정
```

만 분류하는 수준이 아니라,

```json
{
  "불안": 0.81,
  "우울": 0.42,
  "안도": 0.19
}
```

처럼 **감정을 숫자로 표현**하는 방향으로 연구가 진행된다.

예를 들어:

```text
"시험 때문에 너무 우울해요"
→ 우울 0.91

"요즘 살짝 무기력해요"
→ 우울 0.34
```

같은 감정이라도 강도가 다르기 때문에 **0~1 사이 score**로 표현하는 것이 목표이다.

# 3. Regex → BERT → Detection 구조

## 3.1 Regex 단계

### 목적

불필요한 노이즈 제거

Regex는 **문자열 규칙 기반 필터링**이다.

빠르게 데이터를 정리할 수 있다.

### 제거 대상 예시

```text
특수문자 제거
URL 제거
이메일 제거
광고 문구 제거
너무 짧은 문장 제거
욕설 / 금칙어 제거
```

예시:

입력

```text
협찬받아서 작성했어요!!
https://xxxxx.com
```

출력

```text
제거
```

### 특징

장점:

```text
빠름
가벼움
규칙 기반
```

단점:

```text
문맥 이해 불가능
```

예시:

```text
"맛이 미쳤다"

Regex:
긍정인지 부정인지 모름
```

---

## 3.2 BERT 단계

### BERT란?

BERT는 자연어 처리(NLP) 분야에서 많이 쓰이는 모델이다.

정식 명칭:

```text
BERT
=
Bidirectional Encoder Representations from Transformers
```

2018년 Google이 발표했다.

핵심 아이디어:

```text
양방향(Bidirectional)
```

---

## 기존 모델과 차이

기존 모델:

```text
왼쪽 → 오른쪽
```

만 읽었다.

BERT는:

```text
← →
```

앞뒤 문맥을 동시에 본다.

예시:

문장:

```text
나는 오늘 [MASK]을 먹었다
```

BERT는:

```text
"오늘"
"먹었다"
```

를 같이 보고

```text
밥
점심
```

같은 단어를 예측한다.

---

## BERT 핵심 원리

### 1. MLM (Masked Language Model)

문장 일부를 가리고 맞추게 함

예:

```text
나는 [MASK]를 먹었다
```

정답:

```text
밥
```

목적:

```text
문맥 이해 능력 강화
```

---

### 2. NSP (Next Sentence Prediction)

두 문장이 이어지는 관계인지 판단한다.

예:

```text
나는 밥을 먹었다.
그래서 배가 불렀다.
```

→ 관련 있음

---

## BERT 특징

### 1) 양방향성

문장 앞뒤를 동시에 본다.

### 2) Transformer 구조

Attention 기반 구조이다.

BERT는 Transformer의:

```text
Encoder
```

부분만 사용한다.

---

## 기존 모델과 비교

| 모델     | 특징            |
| -------- | --------------- |
| RNN/LSTM | 순차적으로 읽음 |
| GPT      | 한 방향 →       |
| BERT     | 양방향 ← →      |

---

## 전이학습 (Transfer Learning)

BERT는:

```text
사전학습
+
파인튜닝
```

구조이다.

### 1. 사전학습

Google이 이미 수십억 문장으로 학습함

### 2. 파인튜닝

내 데이터로 조금만 추가 학습

예:

```text
감정분석
문서분류
질의응답
```

소량 데이터만 있어도 성능이 높다.

---

## BERT 활용 사례

| 과제        | 설명                  |
| ----------- | --------------------- |
| 감성 분석   | 긍정/부정 분류        |
| 질의응답    | 질문에 맞는 답 찾기   |
| 문서 분류   | 뉴스 자동 분류        |
| 개체명 인식 | 이름·장소·날짜 추출   |
| 검색 개선   | Google 검색 품질 개선 |

---

## GPT vs BERT 차이

### BERT

```text
이해(Encoder)
```

특화

잘하는 것:

```text

분류
검색
감정분석
```

---

### GPT

```text
생성(Decoder)
```

특화

잘하는 것:

```text
글쓰기
요약
챗봇
생성
```

강화학습은:

```text
보상을 최대화하는 방향으로
스스로 학습하는 방식
```

이다.

### 1. Agent

행동하는 주체

예:


환경

예:

```text

```


```text

```

---

텍스트 특화 모델

예:

```text

BERT
Claude

Gemma
Qwen
```

즉,

```text
Foundation Model
⊃
LLM
```
