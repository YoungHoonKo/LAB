# 2. 주요 피드백(Feedback)

## 2.1 긍정적 후기 편향 문제

### 문제점

현재 네이버 블로그 키워드 기반 수집 방식은 다음과 같은 문제가 존재함.

```text
"불안에 좋은 제품"
"우울 완화"
"스트레스에 도움"
```

과 같은 검색어로 데이터를 수집할 경우,

→ **긍정적 후기 중심 데이터만 수집됨**

즉, 실제 사용자 경험 중:

- 효과가 없었던 후기
- 부정적 경험
- 역효과 사례

등이 충분히 반영되지 못하는 **Positive Bias(긍정 편향)** 문제가 발생함.

### 개선 방향

#### 1) 부정 키워드 추가

예시:

```text
불안 악화
효과 없음
실망
도움 안됨
후기 별로
우울 개선 실패
```

#### 2) 중립 키워드 기반 수집

현재:

```text
"불안에 좋은 허브티"
```

↓

변경:

```text
"허브티 후기"
"아로마 오일 사용기"
"스트레스볼 리뷰"
```

→ 이후 LLM이 감정을 판단

즉,

**키워드가 감정을 결정하는 방식이 아니라**
**LLM이 감정을 판별하는 구조로 변경**

## 2.2 Body Text Crawling 문제

### 문제점

현재는 검색 결과 제목 수준만 활용되고 있으며,

실제 핵심 정보는:

> 블로그 내부 본문(body text)

에 존재함.

### 연구 방향

검색 결과 → 하이퍼링크 이동 → 본문 크롤링 구조 필요

Pipeline:

```text
Naver Search API
        ↓
블로그 URL 수집
        ↓
하이퍼링크 이동
        ↓
본문(body text) 추출
        ↓
텍스트 정제
        ↓
LLM 감정 분석
```

### 기술 검토 필요

- iframe 처리
- 모바일 블로그 URL 변환
- CSS selector 기반 body 추출
- HTML cleaning

---

## 2.3 데이터 편향성(Bias)

### 문제점

현재 데이터는 다음 편향 가능성이 존재함.

#### 1. 플랫폼 편향

- 네이버 블로그만 활용

#### 2. 사용자 편향

- 후기 작성 성향이 강한 사용자 중심

#### 3. 감정 편향

- 긍정 후기 과대표집

### 향후 방향

가능하다면 다음 플랫폼 확장 검토

- 네이버 블로그
- 카페
- Reddit
- X(Twitter)
- 뉴스 댓글
- 커뮤니티

---

## 2.4 감정 Labeling 방식 수정

### 기존 방식

연구자가 임의로 정의

예시:

```text
좋음
화남
슬픔
기쁨
```

### 문제점

- 주관적 기준
- 재현성 부족
- 객관성 부족

### 수정 방향

**LLM 기반 감정 추론 방식 적용**

예시:

```text
텍스트 입력
↓
LLM
↓
감정 추론
↓
8개 감정 분류
```

예상 감정 분류:

- 기쁨
- 슬픔
- 분노
- 불안
- 공포
- 놀람
- 혐오
- 안정/평온

즉,

> 사람이 감정을 정하는 방식 → LLM이 감정을 추론하는 방식

으로 변경

# 3. 연구 목표 재정의

## 핵심 패러다임 변화

### 기존 방식 (A → B)

```text
키워드 검색
↓
제품 추천
```

예시:

```text
"불안에 좋은 제품"
```

↓

제품 추천

---

### 목표 방식 (B → A)

```text
후기 데이터
↓
패턴 발견
↓
제품 효과 추론
↓
추천
```

예시:

```text
"이런 후기들이 많이 존재함"
↓
LLM 분석
↓
이 사용자에게 도움될 가능성이 높음
↓
제품 추천
```

즉,

> **후기 기반 역추론(Dictionary Reverse Inference)** 구조

를 목표로 함.

---

## 최종 연구 목표

```text
LLM + 자연어 처리 기반 감성 분석을 통해

대학생의 심리적 상태를 완화할 가능성이 있는
맞춤형 물품 패키지를 추천하는 시스템 구축
```

# 4. 현실적인 연구 방향 (시간 제약 고려)

원래 계획:

```text

↓
추천 시스템 구축
↓
30명 실험
↓
효과 검증

### 단기 목표 수정

↓

↓
Survey 진행
↓
효과 검증
```

즉,

**"실험 먼저 → AI 고도화 나중" 전략**
