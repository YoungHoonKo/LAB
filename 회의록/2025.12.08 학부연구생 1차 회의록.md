### 2.2 데이터 수집 방향

기존 연구 방향은 **재난·질병·기후 변화 상황에서 필요한 물품 추천 시스템 구축**이었다.

이에 따라 다음과 같은 방향으로 데이터를 수집하기로 논의함.

#### 수집 대상 텍스트

- 재난
- 질병
- 기후 변화
- 심리 안정 관련 제품 후기
- 사용자 감정 반응 텍스트

#### 예시 사건 기반 데이터 수집

과거 대규모 재난 상황의 텍스트 데이터를 분석 대상으로 검토

예시:

- 코로나19 (COVID-19)
- 미국 캘리포니아 산불
- 자연재해 및 기후 위기 사례

특히 **코로나 시기(2020년 전후)** 데이터를 확보할 수 있는지 검토 필요

---

### 2.3 데이터 수집 범위 및 API 제한 확인

데이터 수집 시 기간 설정 가능 여부 검토

예시:

- 2024년 ~ 2025년
- 2020년 코로나 시기
- 특정 기간(3개월 단위)

#### 데이터 수집 규모 논의

예시:

- **3개월 단위**
- **100만 건 수준 데이터 수집**
- 총 **300만 건 이상 규모 확장 가능성 검토**

#### API 제한 사항

- **한 번 호출 최대 수집량:** 100개
- **한 검색어(query) 기준 최대 접근 가능 데이터:** 1000개
- **API 호출 1회 = 사용량 1회 차감**

즉, 검색 키워드 다양화가 매우 중요함.

### 2.4 검색 키워드 및 정의 명확화

연구의 재현성과 객관성을 위해 다음 사항을 명확히 정의하기로 함.

#### 1) 검색 키워드 체계화

현재 논의된 키워드 예시:

- 담요
- 디퓨저

- 영양제
- 초콜릿(소형 스낵)
- 호흡 목걸이

→ 추가 키워드 지속 발굴 필요

다음 사항 기록 필요

- 어떤 검색어를 사용했는가

### 2.5 감정 분류(Labeling) 방향

기존의 **긍정 / 부정 이분 분류**를 넘어서,

보다 세밀한 감정 분류를 시도하기로 논의함.

#### 목표 감정 분류 예시 (8개 감정)

- 기쁨
- 슬픔
- 분노
- 불안
- 공포
- 놀람
- 혐오
- 평온/안정

또는 심리 안정 연구 목적에 맞춘 감정 체계 설계 가능

#### Labeling 방향

- 일부 데이터 수동 레이블링 진행
- 제품별 “좋음/나쁨” 샘플 먼저 정의
- 감정 완화 효과 기준 정립

예시:

> 우울감을 완화시키는 제품인가?  
> 스트레스를 낮추는 효과가 있는가?
