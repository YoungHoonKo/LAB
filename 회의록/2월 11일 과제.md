# 1. Problem Definition

본 문서는 학부연구생 첫 번째 과제인 **티백 데이터 기반 LLM 감정분류기 구현** 내용을 정리한 것이다.

목표는 다음과 같다.

CSV에 저장된 블로그 본문 텍스트를 Ollama 로컬 LLM에 입력하여, 각 문장을 **긍정(positive)** 또는 **부정(negative)** 으로 분류하고, 결과를 `output.csv`로 저장한다.  
만약 CSV 안에 정답 라벨(`label`) 컬럼이 존재하면, 모델의 예측 결과와 실제 정답을 비교하여 **Accuracy, Precision, Recall, F1-score, Confusion Matrix**를 계산한다.

즉, 본 과제는 다음 구조를 가진다.

```text
네이버 블로그 데이터 수집
        ↓
본문 텍스트 크롤링
        ↓
텍스트 전처리
        ↓
Ollama 로컬 LLM 입력
        ↓
긍정 / 부정 감정 분류
        ↓
CSV 저장
        ↓
정답 라벨이 있으면 성능 평가
```

# 2. Core Concept

## 2.1 연구 과제명

```text
티백 리뷰 데이터 기반 LLM 감정분류기 구현
```

## 2.2 핵심 목적

본 과제의 핵심 목적은 **티백 관련 블로그 후기 데이터를 수집하고, Ollama 기반 로컬 LLM을 활용하여 해당 텍스트가 긍정 후기인지 부정 후기인지 분류하는 것**이다.

이때 중요한 점은 다음과 같다.

```text
모델 학습 X
파인튜닝 X
LLM 추론 사용 O
Zero-shot Classification O
```

즉, 새로운 모델을 학습시킨 것이 아니라, 이미 학습되어 있는 LLM에게 프롬프트를 통해 다음 작업을 지시한 것이다.

```text
"이 글이 긍정인지 부정인지 판단해줘."
```

따라서 이 방식은 **Inference 기반 감정 분석 시스템**이며, 더 구체적으로는 **Zero-shot Sentiment Classification**이라고 볼 수 있다.

# 4. Solution

# LLM 기반 감정 파이프라인 코드

```python
import re
import json
import time
import requests
import pandas as pd
from tqdm import tqdm

try:
    from sklearn.metrics import accuracy_score, classification_report, confusion_matrix
except ImportError:
    accuracy_score = None

OLLAMA_URL = "http://localhost:11434/api/generate"

MODEL = "llama3.1:8b"
INPUT_CSV = "input.csv"
OUTPUT_CSV = "output.csv"

TEXT_COL = "text"
LABEL_COL = "label"

PROMPT_TEMPLATE = """너는 감정분류기다.
아래 문장을 "긍정" 또는 "부정" 중 하나로만 분류해.
다른 설명은 절대 하지 마. 출력은 반드시 한 단어만: 긍정 또는 부정.
애매하면 부정.

문장: {text}
출력:
"""

def normalize_label(x: str) -> str:
    """모델 출력/정답 라벨을 '긍정'/'부정'으로 통일"""
    if x is None:
        return ""
    s = str(x).strip()

    s_low = s.lower()
    if "긍정" in s or "positive" in s_low or s_low in ["pos", "1", "p"]:
        return "긍정"
    if "부정" in s or "negative" in s_low or s_low in ["neg", "0", "n"]:
        return "부정"

    if re.search(r"긍정", s):
        return "긍정"
    if re.search(r"부정", s):
        return "부정"
    return ""

def call_ollama(model: str, prompt: str, max_retries: int = 3) -> str:
    payload = {
        "model": model,
        "prompt": prompt,
        "stream": False,
        "options": {
            "temperature": 0.0,
            "top_p": 1.0
        }
    }

    for attempt in range(1, max_retries + 1):
        try:
            r = requests.post(OLLAMA_URL, json=payload, timeout=120)
            r.raise_for_status()
            data = r.json()
            return data.get("response", "").strip()
        except Exception as e:
            if attempt == max_retries:
                raise
            time.sleep(1.5 * attempt)

def main():
    df = pd.read_csv(INPUT_CSV)

    if TEXT_COL not in df.columns:
        raise ValueError(f"CSV에 '{TEXT_COL}' 컬럼이 없어. 현재 컬럼: {list(df.columns)}")

    preds = []
    for text in tqdm(df[TEXT_COL].fillna("").astype(str).tolist(), desc="Classifying"):
        prompt = PROMPT_TEMPLATE.format(text=text)
        raw = call_ollama(MODEL, prompt)
        pred = normalize_label(raw)

        if pred == "":
            pred = "부정"
        preds.append(pred)

    df["pred"] = preds
    df.to_csv(OUTPUT_CSV, index=False, encoding="utf-8-sig")
    print(f"\n저장 완료: {OUTPUT_CSV}")

    if LABEL_COL in df.columns and accuracy_score is not None:
        gold = df[LABEL_COL].apply(normalize_label)
        valid_mask = gold.isin(["긍정", "부정"])
        if valid_mask.sum() == 0:
            print("\nlabel 컬럼은 있지만 '긍정/부정'으로 정규화 가능한 값이 없어 평가를 건너뜀.")
            return

        y_true = gold[valid_mask].tolist()
        y_pred = df.loc[valid_mask, "pred"].tolist()

        acc = accuracy_score(y_true, y_pred)
        print(f"\nAccuracy: {acc:.4f}")
        print("\nClassification report:")
        print(classification_report(y_true, y_pred, labels=["긍정", "부정"]))
        print("\nConfusion matrix (rows=true, cols=pred):")
        print(confusion_matrix(y_true, y_pred, labels=["긍정", "부정"]))
    else:
        print("\nlabel 컬럼이 없어서 또는 sklearn 미설치로 정확도 평가는 생략함.")
        print("정확도를 계산하려면 input.csv에 label 컬럼을 추가해야 함.")

if __name__ == "__main__":
    main()
```

## 5.7 normalize_label 함수

```python
def normalize_label(x: str) -> str:
```

이 문법은 타입 힌트이다.

```text
x: str
```

은 `x`가 문자열일 가능성이 높다는 표시이다.

```text
-> str
```

은 이 함수가 문자열을 반환한다는 표시이다.

즉, 의미는 다음과 같다.

```text

```

중요한 점은 타입 힌트는 실행을 강제하지 않는다.  
코드를 읽는 사람이 더 쉽게 이해하도록 돕는 표시이다.

## 5.8 normalize_label 함수의 역할

LLM이 항상 정확히 `긍정` 또는 `부정`만 출력한다는 보장은 없다.

예를 들어 다음처럼 출력할 수 있다.

```text
긍정입니다.
positive
POS
1
이 문장은 긍정적인 후기입니다.
```

따라서 이를 하나의 표준 라벨로 통일해야 한다.

```text
positive, POS, 1, 긍정입니다 → 긍정
negative, NEG, 0, 부정입니다 → 부정
```

이 과정을 정규화(normalization)라고 한다.

## 5.9 정규표현식과 raw string

```python
if re.search(r"긍정", s):
    return "긍정"
```

`re.search()`는 문자열 안에 특정 패턴이 있는지 찾는 함수이다.

```text
re.search(패턴, 문자열)
```

예시:

```python
re.search(r"긍정", "이 문장은 긍정입니다")
```

결과적으로 `"긍정"`이라는 단어가 포함되어 있으므로 매칭된다.

여기서 `r"긍정"`의 `r`은 raw string을 의미한다.

raw string은 백슬래시(`\`)를 특수문자로 해석하지 않고 그대로 처리한다.  
정규표현식에서는 `\d`, `\s`, `\w` 같은 표현을 많이 사용하므로 raw string을 자주 사용한다.

예시:

```python
r"\d+"
```

의미:

```text
숫자가 1개 이상 반복되는 패턴
```

## 5.12 stream이란?

스트리밍(streaming)은 모델의 답변을 한 번에 받는 것이 아니라, 조금씩 나누어 받는 방식이다.

예시:

```text
스
스트
스트레
스트레스
스트레스 완화
```

이 코드에서는 감정분류 결과가 짧기 때문에 한 번에 받는 것이 더 단순하다.

```python
"stream": False
```

의미:

```text
결과를 한 번에 받아라.
```

---

## 5.13 temperature란?

`temperature`는 LLM 출력의 랜덤성을 조절하는 값이다.

| 값   | 의미                        |
| ---- | --------------------------- |
| 낮음 | 더 일관적이고 보수적인 출력 |
| 높음 | 더 다양하고 창의적인 출력   |

## 5.32 Precision, Recall, F1-score

`classification_report()`는 다음 지표를 출력한다.

| 지표      | 의미                                            |
| --------- | ----------------------------------------------- |
| Precision | 모델이 긍정이라고 예측한 것 중 실제 긍정인 비율 |
| Recall    | 실제 긍정 중 모델이 긍정으로 맞춘 비율          |
| F1-score  | Precision과 Recall의 조화평균                   |
| Support   | 해당 라벨의 실제 데이터 개수                    |

### Precision

```text
Precision = TP / (TP + FP)
```

의미:

```text
모델이 긍정이라고 한 것 중 진짜 긍정은 얼마나 되는가?
```

### Recall

```text
Recall = TP / (TP + FN)
```

의미:

```text
실제 긍정 중 모델이 얼마나 잘 찾아냈는가?
```

### F1-score

```text
F1 = 2 × Precision × Recall / (Precision + Recall)
```

의미:

```text
Precision과 Recall을 균형 있게 반영한 점수
```

# 6. Inference, Training, Fine-tuning 차이

## 6.1 Inference

Inference는 이미 학습된 모델을 사용하여 결과를 예측하는 과정이다.

본 과제에서는 Ollama에 설치된 LLM에게 다음처럼 지시한다.

```text
"이 문장을 긍정 또는 부정으로 분류해."
```

모델은 이미 학습되어 있으므로 파라미터를 바꾸지 않는다.

본 과제는 여기에 해당한다.

```text
Ollama LLM + Prompt → 감정분류 결과
```

---

## 6.2 Training

Training은 모델이 데이터를 보고 파라미터를 직접 조정하는 과정이다.

예시:

```text
입력: 이 제품 진짜 좋아요
정답: 긍정
```

이런 데이터를 수천 개 이상 넣고, 모델이 오차를 줄이도록 가중치를 업데이트한다.

본 과제에서는 Training을 하지 않았다.

---

## 6.3 Fine-tuning

Fine-tuning은 이미 학습된 모델을 특정 작업에 맞게 추가 학습시키는 과정이다.

예시:

```text
기본 Llama 모델
        ↓
한국어 후기 데이터 10,000개 추가 학습
        ↓
후기 감정분류 특화 모델
```

본 과제에서는 Fine-tuning을 하지 않았다.

# 9. Limitations

현재 구현 방식의 한계는 다음과 같다.

1. 긍정/부정 이진 분류만 수행한다.
2. LLM의 출력이 항상 안정적이라는 보장은 없다.
3. 프롬프트 설계에 따라 결과가 달라질 수 있다.
4. 정답 라벨이 없는 경우 정확도 평가가 불가능하다.
5. 네이버 블로그 데이터는 긍정 후기 중심으로 편향될 가능성이 있다.
6. 본 방식은 파인튜닝이 아니므로 특정 도메인에 완전히 최적화된 모델은 아니다.

```text
제품 후기 데이터 수집
        ↓
본문 크롤링
        ↓
LLM 감정분류기
        ↓
물품별 감성지수 계산
        ↓

LLM 기반 감성지수 분석을 활용한 맞춤형 심리 안정 키트 추천 시스템
```

을 구현하기 위한 첫 번째 핵심 모듈이다.
