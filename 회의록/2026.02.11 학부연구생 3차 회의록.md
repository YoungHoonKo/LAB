# 2. 주요 논의 내용

## 2.1 현재 데이터셋 활용 방향

### 논의 사항

현재까지 수집한 **3개의 데이터셋(Input Data)** 을 활용하여 어떤 연구 결과물을 만들 수 있는지 논의함.

현재 데이터셋은:

```text
네이버 블로그 후기 데이터
제품 후기 데이터
감정 관련 텍스트 데이터
```

로 구성되어 있으며,

이는 **LLM 학습/추론을 위한 Input 데이터 역할**을 수행함.

### 핵심 방향

현재 단계는:

```text
데이터 수집
↓
자연어 처리
↓
감정 추론
```

이며,

이후 발전 방향으로는:

```text
감정분류기
↓
추천 시스템
↓
챗봇(Chatbot)
```

까지 확장 가능성을 고려함.

## 2.3 LLM을 실제 어떻게 활용할 것인가?

### 핵심 질문

> **“LLM이 문맥(Context)을 어떻게 이해하는가?”**

단순 키워드 매칭이 아니라,

문맥 기반 자연어 처리(Context Understanding)를 목표로 함.

예시:

### 사례 1

```text
"비타민 먹고 활력이 생겼다."
```

LLM 해석:

```text
감정 → 긍정
효과 → 활력 증가
상태 변화 → 에너지 상승
```

---

### 사례 2

```text
"노란 조명을 사용했더니 차분해졌다."
```

LLM 해석:

```text
감정 → 안정감
효과 → 심리적 진정
```

---

### 사례 3

```text
"가중 담요를 쓰니 안정감이 생겼다."
```

LLM 해석:

```text
감정 → 심리 안정
효과 → 불안 완화 가능성
```

즉,

> **문맥(Context)을 따라 자연어를 이해하고 감정을 분류하는 구조**

를 목표로 함.

---

## 2.4 현재 우리가 만든 것

현재까지 수행한 작업은:

> **LLM이 처리할 Input 데이터를 구축한 단계**

즉,

```text
네이버 블로그
↓
텍스트 수집
↓
데이터셋 구축
```

까지 진행된 상태임.

이제부터는:

```text
Input Data
↓
LLM
↓
감정 분류(Output)
```

단계를 개발해야 함.

---

## 2.5 Output 정의

### 최종적으로 가능한 Output

#### 1단계 (현재 목표)

```text
감정
```

예시:

- 긍정
- 부정

#### 2단계 (향후 확장)

```text
감정 + 추천 상품
```

예시:

```text
감정: 불안

추천:
- 아로마 오일
- 허브티
- 스트레스볼
```

#### 3단계 (장기 목표)

```text
챗봇(Chatbot)
```

사용자와 대화하며:

- 감정 분석
- 심리 상태 추론
- 맞춤형 추천

까지 수행

## 2.6 감정분류기 방향

### 목표

**LLM 기반 감정분류기 개발**

현재는 복잡한 다중 감정보다,

우선 **Binary Classification(이진 분류)** 로 진행

### Output

```text

```

### Input

```text

```

### 구조

```text
Naver Blog Text
        ↓
Preprocessing
        ↓
LLM
        ↓
Positive / Negative
```

## 2.7 챗봇(Chatbot) 가능성

회의 중 추가 논의:

향후 발전 방향으로

> **LLM 기반 챗봇(Chatbot)**

을 고려할 수 있음.

