# 2026.03.11 학부연구생 회의록

## 1. Problem Definition

이번 회의의 핵심은 기존 **Ollama 기반 긍정/부정 감정분류기**를 확장하여, **LangChain·RAG 기반 데이터 처리 구조**, **8개 감정 세부 분류**, **광고성 데이터 제거**, **웹 서비스 구조 설계**로 발전시키는 것이다.

---

## 2. Core Concept

이번 회의에서 논의된 핵심 개념은 다음과 같다.

|   구분    | 내용                                                       |
| :-------: | :--------------------------------------------------------- |
|  Ollama   | 로컬 환경에서 LLM을 실행하기 위한 도구                     |
| LangChain | LLM 애플리케이션 개발을 위한 프레임워크                    |
|    RAG    | 외부 데이터를 검색한 뒤 LLM 답변에 활용하는 구조           |
| 감정분류  | 기존 긍정/부정에서 8개 세부 감정으로 확장                  |
| 광고 제거 | 블로그 데이터 중 홍보성 글을 필터링하는 전처리             |
| 웹페이지  | 사용자가 감정 문장을 입력하면 추천 결과가 나오는 화면 설계 |

---

## 3. Logical Analysis

### 3.1 기존 연구 흐름

기존에는 다음 구조였다.

```text
네이버 블로그 데이터 수집
        ↓
본문 크롤링
        ↓
Ollama LLM 입력
        ↓
긍정 / 부정 분류
```

### 3.2 확장 방향

이번 회의 이후 목표 구조는 다음과 같다.

```text
웹 데이터 수집
        ↓
광고성 데이터 제거
        ↓
LangChain + RAG 기반 데이터 검색
        ↓
Ollama / GPT / Gemini / Qwen / Gemma 비교
        ↓
8개 감정 세부 분류
        ↓
사용자 감정 입력 기반 키트 추천
        ↓
웹페이지 출력
```

### 3.3 감정 분류 확장

기존 출력:

```text
긍정 / 부정
```

확장 출력:

```text
기쁨
슬픔
분노
불안
공포
놀람
혐오
평온
```

이 방식은 단순 이진 분류보다 사용자의 심리 상태를 더 세밀하게 반영할 수 있다.

### 3.4 광고성 데이터 제거 필요성

네이버 블로그 데이터에는 체험단, 협찬, 구매 유도, 링크 삽입 등 광고성 글이 포함될 가능성이 있다.

이러한 데이터가 많으면 감정분류 결과와 추천 결과가 편향될 수 있다.

따라서 다음 기준을 검토해야 한다.

```text
협찬 / 광고 / 제공받아 / 체험단 / 구매링크 / 최저가 / 스마트스토어 / 쿠팡파트너스
```

해당 표현이 포함된 글은 광고성 데이터로 분류하거나 별도 태그를 붙일 수 있다.

---

## 4. 회의 정리

### 4.1 연구 방향

이번 회의에서 결정된 연구 방향은 다음과 같다.

1. Ollama 기반 LLM 감정분류기를 계속 활용한다.
2. LangChain과 RAG를 사용해 데이터 검색 및 활용 구조를 고도화한다.
3. 긍정/부정 분류를 넘어 8개 감정으로 세분화한다.
4. 광고성 블로그 데이터를 제거하거나 별도 분류한다.
5. 여러 LLM 모델의 감정분류 결과를 비교한다.
6. 사용자가 감정을 입력하면 키트 추천 결과가 나오는 웹페이지를 설계한다.
