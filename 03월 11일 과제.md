## 1. Problem Definition

이번 과제의 핵심은 기존 **네이버 블로그 URL 수집 → 본문 크롤링 → Ollama 기반 감정분류** 구조를 확장하여, 실제 사용자 후기 데이터를 기반으로 **심리 안정 케어 키트 추천 시스템**을 설계하는 것이다.

최종 목표는 다음과 같다.

```text
사용자가 "나 불안하고 마음이 너무 힘들어"와 같은 문장을 입력
        ↓
LLM이 사용자의 상태와 감정을 분석
        ↓
RAG가 증상 DB와 제품 후기 데이터를 검색
        ↓
LLM이 적절한 제품 조합을 생성
        ↓
사용자에게 맞춤형 케어 키트 추천
```

# 3.2 파이프라인

## Step 1 — URL 수집 (`step1_collect.py`)

네이버 API로 키워드를 검색해서 블로그 URL을 수집한다.

각 키워드마다 검색 결과를 순서대로 확인하고, `title` 또는 `description`에 캐모마일 관련 단어가 있으면 관련 있는 글로 판단한다.

관련 없는 글이 연속으로 5개 나오면 해당 키워드의 수집을 중단한다.

관련 있는 글이 다시 나오면 연속 무관 카운터는 0으로 초기화된다.

저장 파일:

```text
step1_캐모마일.csv
log_step1_캐모마일.csv
```

---

## Step 2 — 메타데이터 필터 (`step2_filter.py`)

크롤링 전에 API 메타데이터만으로 빠르게 1차 필터링한다.

필터링 기준:

```text
1. title/description에 광고 문구가 있으면 제거
2. 동일 블로거가 5개 초과 포스트를 올렸으면 광고 계정 가능성으로 제거
3. 중복 URL 재확인
```

광고 의심 키워드:

```text
협찬
광고
체험단
제공받아
원고료
구매링크
스마트스토어
쿠팡파트너스
최저가
할인코드
```

저장 파일:

```text
step2_캐모마일.csv
log_step2_캐모마일.csv
```

---

## Step 3 — 본문 크롤링 + 2차 필터 (`step3_crawl.py`)

실제 블로그 본문을 크롤링하고, 내용 기반으로 다시 필터링한다.

필터링 기준:

```text
1. 본문에서 광고 문구 2차 확인 → 있으면 제거
2. 본문 50자 미만 → 제거
3. 본문에 "캐모마일" 관련 표현 없음 → 제거
4. 직접 후기 점수 2점 미만 → 제거
5. 부정 표현 태깅 → is_negative 컬럼 추가
```

직접 후기 점수 기준:

```text
1인칭 표현: 저는, 마셨어요 등              +2점
기간/횟수: 일주일째, 매일 등               +2점
신체/감정 반응: 긴장이 풀렸어요 등          +2점
구매 경험: 샀어요, 주문했어요 등            +1점
맛/향 묘사: 달콤한, 은은한 등               +1점

통과 기준: 2점 이상
```

저장 파일:

```text
step3_캐모마일.csv
log_step3_캐모마일.csv
```

---

## Step 4 — Ollama 분석 (`step4_analyze.py`)

Ollama 로컬 LLM 또는 EEVE-Korean 모델을 사용하여 블로그 본문마다 세 가지 정보를 추출한다.

출력 JSON 구조:

```json
{
  "states": ["불안", "긴장"],
  "effect": "긍정",
  "emotion": "안도"
}
```

각 필드 의미:

```text
states: 사용자가 제품을 사용한 이유 또는 상태
effect: 제품 사용 효과 평가
emotion: 제품 사용 후 감정
```

JSON 파싱에 실패하거나 유효하지 않은 값이 나오면 자동 보정한다.

저장 파일:

```text
final_캐모마일.csv
log_step4_캐모마일.csv
```

# 3.4 생성형 AI에게 질문한 프로젝트 문제

## 연구 목적

네이버 API를 사용하여 블로그 URL을 CSV로 수집한다.

연구 목적은 네이버 블로그의 실제 사용자 후기 데이터를 바탕으로, 사용자가 다음과 같이 입력했을 때 맞춤형 제품을 추천하는 것이다.

```text
나 불안하고 마음이 너무 힘들어
```

시스템은 사용자의 감정 상태에 맞는 제품을 추천하고, 이를 기반으로 케어 키트를 구성한다.

## 현재 고민

```text
1. 광고성 블로그를 어떻게 제거할 것인가?
2. 키워드와 무관한 이상한 블로그를 어떻게 제거할 것인가?
3. 키워드를 어떻게 설정해야 편향을 줄일 수 있는가?
4. "캐모마일 불안 완화 효과"처럼 긍정 편향 키워드를 쓰면 긍정 후기만 모이지 않는가?
```

# 3.5 코드를 작성하면서 생각한 문제점

## 문제 1. Step 1에서 연속 5개 이상 무관하면 중단하는 구조

질문:

```text
연속으로 무관한 글이 5개 나오면 중단하는데,
그전에 이미 수집된 관련 없는 글은 제거되는가?
```

정리:

현재 코드에서는 `is_relevant()`를 통과한 글만 `all_items`에 저장된다.  
따라서 `is_relevant()`가 정확하다면 관련 없는 글은 저장되지 않는다.

다만 `title/description`에 캐모마일이라는 단어가 있지만 실제 본문은 다른 내용일 수 있으므로, Step 3 본문 필터가 필요하다.

5. 관련 없는 글이 연속 5개 나오면 해당 키워드 수집 중단
6. 관련 있는 글이 나오면 카운터를 0으로 초기화

# 3.10 Qwen vs GPT vs Gemini vs Gemma 비교

## 모델 유형

| 아이콘 | 이름       | 특징                     |
| ------ | ---------- | ------------------------ |
| 🔶      | Pretrained | 기초 학습만 된 원본 모델 |
| 💬      | Chat       | 대화 최적화 모델         |

| ❓      | Fine-tuned | 특정 목적 추가학습 모델  |

## 벤치마크 의미

벤치마크는 AI 모델을 공정하게 비교하기 위한 표준 시험이다.

| 컬럼     | 의미                  | 쉬운 설명                |
| -------- | --------------------- | ------------------------ |
| Average  | 전체 평균 점수        | 종합 성적                |
| IFEval   | Instruction Following | 지시를 잘 따르는지       |
| BBH      | Big Bench Hard        | 복잡한 논리 추론         |
| MATH     | Math                  | 수학 문제 풀이           |
| GPQA     | Graduate-Level QA     | 전문가 수준 지식         |
| MUSR     | Multi-Step Reasoning  | 긴 문맥 기반 다단계 추론 |
| MMLU     | Massive Multitask     | 다양한 분야 지식         |
| CO₂ Cost | 환경 비용             | 모델 평가 시 탄소 비용   |

## 모델별 예상 비교

# 4. Solution

## Step 1 URL 수집 코드

주의:

아래 코드는 구조 설명용이다.  
GitHub에 업로드할 때는 `CLIENT_ID`, `CLIENT_SECRET`을 코드에 직접 적으면 안 된다.  
반드시 환경변수로 관리해야 한다.

```python
import os
import requests
import pandas as pd
import time
import re
from datetime import datetime, timedelta

CLIENT_ID = os.getenv("NAVER_CLIENT_ID")
CLIENT_SECRET = os.getenv("NAVER_CLIENT_SECRET")

PRODUCT = "캐모마일"

KEYWORDS = [
    # 중립 후기
    "캐모마일 후기",
    "캐모마일 티백 후기",
    "캐모마일 마셔봤는데",
    "캐모마일 먹어봤어",
    "카모마일 후기",
    "케모마일 후기",

    # 증상 중심
    "캐모마일 불안",
    "캐모마일 긴장",
    "캐모마일 우울",

    # 사용 경험
    "캐모마일 효과",
    "캐모마일 차 마시고",

    # 부정 후기
    "캐모마일 실망",
    "캐모마일 별로",
]

RELEVANCE_KEYWORDS = [
    "캐모마일",
    "카모마일",
    "케모마일",
    "캐모밀",
    "카모밀",
    "chamomile",
    "camomile",
]

CONSECUTIVE_IRRELEVANT_LIMIT = 5
DATE_FILTER_YEARS = 0

def search_naver_blog(query, display=100, start=1):
    url = "https://openapi.naver.com/v1/search/blog.json"
    headers = {
        "X-Naver-Client-Id": CLIENT_ID,
        "X-Naver-Client-Secret": CLIENT_SECRET,
    }
    params = {
        "query": query,
        "display": display,
        "start": start,
        "sort": "sim",
    }

    try:
        res = requests.get(url, headers=headers, params=params, timeout=10)

        if res.status_code == 200:
            return res.json().get("items", [])

        print(f"API 오류: {res.status_code}")

    except Exception as e:
        print(f"요청 실패: {e}")

    return []

def is_within_date_range(postdate_str: str) -> bool:
    if not DATE_FILTER_YEARS:
        return True

    try:
        post_dt = datetime.strptime(str(postdate_str), "%Y%m%d")
        cutoff = datetime.now() - timedelta(days=365 * DATE_FILTER_YEARS)
        return post_dt >= cutoff

    except Exception:
        return True

def clean_html(text: str) -> str:
    return re.sub(r"<.*?>", "", str(text))

def is_relevant(item: dict) -> bool:
    title = clean_html(item.get("title", ""))
    desc = clean_html(item.get("description", ""))

    return any(
        keyword.lower() in title.lower() or keyword.lower() in desc.lower()
        for keyword in RELEVANCE_KEYWORDS
    )

def collect_urls(keywords):
    all_items = {}
    kw_log = []

    for kw in keywords:
        print(f"\\n[{PRODUCT}] '{kw}' 수집 중...")

        collected = 0
        consecutive_irrelevant = 0
        stopped_early = False

        for start in range(1, 1000, 100):
            items = search_naver_blog(kw, display=100, start=start)

            if not items:
                break

            for item in items:
                link = item.get("link", "")

                if "blog.naver.com" not in link:
                    continue

                if not is_within_date_range(item.get("postdate", "")):
                    continue

                if is_relevant(item):
                    if link not in all_items:
                        all_items[link] = {
                            "url": link,
                            "title": clean_html(item.get("title", "")),
                            "description": clean_html(item.get("description", "")),
                            "bloggername": item.get("bloggername", ""),
                            "postdate": item.get("postdate", ""),
                            "product": PRODUCT,
                            "keyword": kw,
                        }

                        collected += 1

                    consecutive_irrelevant = 0

                else:
                    consecutive_irrelevant += 1

                    if consecutive_irrelevant >= CONSECUTIVE_IRRELEVANT_LIMIT:
                        print(
                            f"연속 {CONSECUTIVE_IRRELEVANT_LIMIT}개 무관련, "
                            f"수집 중단 (수집: {collected}개)"
                        )
                        stopped_early = True
                        break

            if stopped_early:
                break

            time.sleep(0.1)

        status = "중단(연속무관련)" if stopped_early else "완료"
        print(f"[{status}] '{kw}': {collected}개 수집")

        kw_log.append({
            "keyword": kw,
            "status": status,
            "collected": collected,
        })

    return list(all_items.values()), kw_log

if __name__ == "__main__":
    print("=" * 50)
    print(f"STEP 1: {PRODUCT} URL 수집")
    print("=" * 50)

    items, kw_log = collect_urls(KEYWORDS)

    pd.DataFrame(items).to_csv(
        "step1_캐모마일.csv",
        index=False,
        encoding="utf-8-sig"
    )

    pd.DataFrame(kw_log).to_csv(
        "log_step1_캐모마일.csv",
        index=False,
        encoding="utf-8-sig"
    )

    print(f"\\n저장 완료: step1_캐모마일.csv ({len(items)}개)")
    print("로그 저장: log_step1_캐모마일.csv")
    print("\\n[키워드별 수집량]")
    print(pd.DataFrame(kw_log).to_string(index=False))
```

# 5. Limitations

현재 정리된 내용의 한계는 다음과 같다.

```text
1. Step 1은 title/description 기반 관련성 판단이므로 본문 내용까지 보장하지 않는다.
2. 광고성 블로그 제거는 키워드 기반이므로 완벽하지 않다.
3. 캐모마일 관련 후기 중 맛 후기와 감정 변화 후기를 구분하는 기준이 아직 실험적으로 검증되지 않았다.
4. Ollama 감정 분류 결과는 모델과 프롬프트에 따라 달라질 수 있다.
5. Qwen, GPT, Gemini, Gemma 비교는 동일 데이터셋 기반 실험이 필요하다.
6. 키트 구성의 실제 심리 안정 효과는 설문조사로 검증해야 한다.
7. 세인트존스워트, 비타민 D 등 영양제는 개인 건강 상태나 복용 약물에 따라 주의가 필요하므로 일반 추천 시 별도 안내가 필요하다.
```
